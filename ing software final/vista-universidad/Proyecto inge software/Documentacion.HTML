1. Introducción

El presente documento detalla el desarrollo y las características del proyecto de mejora visual y funcional para el portal estudiantil de la Universidad San Pablo. El objetivo principal fue modernizar la interfaz de usuario, unificar el diseño a través de las diferentes secciones y mejorar la experiencia general del estudiante al interactuar con el portal.

Se ha puesto énfasis en la claridad de la información, la facilidad de navegación y la implementación de un diseño responsivo utilizando el framework Bootstrap 5 para las vistas internas del portal, manteniendo un diseño personalizado para el login y la recuperación de contraseña. La persistencia de datos, como la información de perfil y el estado de inicio de sesión, se maneja a través de localStorage del navegador.

2. Tecnologías Utilizadas

  • HTML5: Para la estructura semántica del contenido
  • CSS3: Para la estilización visual, incluyendo variables CSS, Flexbox y Grid
  • Bootstrap 5.3.0: Framework CSS utilizado para el diseño responsivo y componentes UI en la mayoría de las vistas internas (post-login)
  • JavaScript (ES6+): Para la interactividad del lado del cliente, manejo de formularios, manipulación del DOM y gestión de datos con localStorage
  • Font Awesome 6.4.0: Para la iconografía utilizada en toda la aplicación
  • Google Fonts (Poppins): Tipografía principal utilizada en el panel de main.html para una apariencia moderna

4. Descripción de Vistas y Casos de Uso

4.1. index.html (Página de Inicio de Sesión)

Propósito:

Permite a los usuarios autenticarse para acceder al portal estudiantil.

Elementos Clave:

  • Logo de la Universidad
  • Formulario con campos para "Usuario" y "Contraseña"
  • Checkbox "No cerrar sesión"
  • Botón "Iniciar"
  • Enlace para recuperación de contraseña

Estilos y Scripts:

  • Estilos definidos en styles/styles.css
  • Lógica de autenticación manejada por scripts/login.js (guarda en localStorage)

Casos de Uso Principales:

  • CASO DE USO-01: Inicio de Sesión Exitoso
    • Actor: Usuario
    • Descripción: El usuario ingresa credenciales válidas y accede al panel principal
    • Flujo Principal:
      1. El usuario ingresa su nombre de usuario en el campo "Usuario"
      2. El usuario ingresa su contraseña en el campo "Contraseña"
      3. (Opcional) El usuario marca la casilla "No cerrar sesión"
      4. El usuario hace clic en el botón "Iniciar"
      5. El sistema valida las credenciales (simulado)
      6. El sistema guarda la información de sesión en localStorage
      7. El sistema redirige al usuario a main.html
  • CASO DE USO-02: Intento de Inicio de Sesión con Campos Vacíos
    • Actor: Usuario
    • Descripción: El usuario intenta iniciar sesión sin ingresar credenciales
    • Flujo Principal:
      1. El usuario deja uno o ambos campos ("Usuario", "Contraseña") vacíos
      2. El usuario hace clic en el botón "Iniciar"
      3. El sistema (según login.js) no procede con el guardado en localStorage ni la redirección si los campos están vacíos (basado en username && password). Se requiere validación HTML required en los inputs para una mejor UX
  • CASO DE USO-03: Acceder a Recuperación de Contraseña
    • Actor: Usuario
    • Descripción: El usuario navega a la página de recuperación de contraseña
    • Flujo Principal:
      1. El usuario hace clic en el enlace "¿Olvidé mi contraseña?"
      2. El sistema redirige al usuario a recuperar_contraseña.html

Elementos Clave:

  • Formulario con campo para "Correo electrónico registrado"
  • Botón "Enviar instrucciones"
  • Enlace "Volver al inicio"

Estilos y Scripts:

  • Estilos definidos en styles/styles.css
  • Lógica (hipotética) manejada por scripts/recover.js (archivo no proporcionado)

Casos de Uso Principales:

  • CASO DE USO-01: Solicitar Recuperación de Contraseña
    • Actor: Usuario
    • Descripción: El usuario ingresa su correo electrónico para recibir instrucciones de recuperación
    • Flujo Principal:
      1. El usuario ingresa su correo electrónico en el campo correspondiente
      2. El usuario hace clic en el botón "Enviar instrucciones"
      3. El sistema (hipotéticamente a través de recover.js) procesa la solicitud. (Ej. Muestra un mensaje de confirmación)
  • CASO DE USO-02: Volver a Inicio de Sesión
    • Actor: Usuario
    • Descripción: El usuario regresa a la página de inicio de sesión
    • Flujo Principal:
      1. El usuario hace clic en el enlace "Volver al inicio"
      2. El sistema redirige al usuario a index.html

4.3. main.html (Panel Principal / Dashboard)

Propósito:

Actúa como el centro de navegación principal después del inicio de sesión, mostrando información relevante y accesos directos.

Elementos Clave:

  • Header: Logo, saludo al usuario (enlazado a perfil) y botón de cerrar sesión
  • Sidebar: Buscador, menú de navegación (Inicio, Asignaciones, Cursos, Cuenta Corriente, Perfil de Usuario)
  • Content Area:
    • Slider visual de imágenes (simulado)
    • Tarjeta de información de pagos
    • Sección de "Avisos Importantes" con varias tarjetas de noticias/alertas
  • Footer: Información de copyright

Estilos y Scripts:

  • Estilos integrados en la etiqueta <style></style> (originalmente en styles/stylesMain.css)
  • Utiliza Font Awesome para iconos y Google Fonts (Poppins)
  • Incluye scripts/login.js (para checkLogin() y logout())
  • Script inline para actualizar el año del footer y mostrar el nombre de usuario

Casos de Uso Principales:

  • CASO DE USO-01: Visualizar Panel Principal
    • Actor: Usuario (Estudiante)
    • Descripción: El usuario ve la información general del dashboard, incluyendo avisos y el slider
    • Flujo Principal:
      1. El usuario accede a main.html (después de un login exitoso)
      2. El sistema muestra el header con el saludo y el logo
      3. El sistema muestra la barra lateral con opciones de navegación
      4. El sistema muestra el slider de imágenes, la información de pagos y los avisos importantes
  • CASO DE USO-02: Navegar a una Sección desde Sidebar
    • Actor: Usuario
    • Descripción: El usuario selecciona una opción del menú lateral para ir a otra sección del portal
    • Flujo Principal:
      1. El usuario hace clic en un enlace del menú lateral (ej. "Cursos", "Cuenta Corriente", "Perfil de Usuario")
      2. El sistema redirige al usuario a la página HTML correspondiente (ej. cursos.html)
  • CASO DE USO-03: Navegar al Perfil de Usuario desde Header
    • Actor: Usuario
    • Descripción: El usuario hace clic en el saludo de bienvenida para acceder a su perfil
    • Flujo Principal:
      1. El usuario hace clic en el texto "Bienvenido, [NombreUsuario]" en el header
      2. El sistema redirige al usuario a perfil_usuario.html
  • CASO DE USO-04: Cerrar Sesión
    • Actor: Usuario
    • Descripción: El usuario finaliza su sesión en el portal
    • Flujo Principal:
      1. El usuario hace clic en el botón "Cerrar Sesión" (icono de salida) en el header
      2. El sistema ejecuta la función logout(), que elimina los datos de sesión de localStorage
      3. El sistema redirige al usuario a index.html

    • Flujo Principal:

      2. El sistema ejecuta loadUserData(), poblando los campos del formulario y la imagen de perfil con los datos de localStorage
  • CU-EDITUSR-02: Modificar Datos Personales y Guardar
    • Actor: Usuario
    • Descripción: El usuario actualiza uno o más campos de su información personal y/o su foto de perfil
    • Flujo Principal:
      1. El usuario modifica los valores en los campos del formulario
      2. (Opcional) El usuario selecciona una nueva imagen de perfil, la cual se previsualiza
      3. El usuario hace clic en "Guardar Cambios"
      4. El sistema ejecuta saveUserData(), que recopila los datos del formulario, convierte la imagen a Data URL (si es nueva), guarda el objeto actualizado en localStorage y redirige a perfil_usuario.html
  • CU-EDITUSR-03: Cancelar Edición
    • Actor: Usuario
    • Descripción: El usuario decide no guardar los cambios y volver al perfil
    • Flujo Principal:
      1. El usuario hace clic en el botón "Cancelar"
      2. El sistema redirige al usuario a perfil_usuario.html sin guardar cambios

Propósito:

Para las vistas main.html, cuenta_corriente.html, cursos.html, asignaciones.html, estado_cuenta.html, pago_tarjeta.html, perfil_usuario.html y todas sus sub-páginas de edición (editar-*.html), los estilos principales se basan en Bootstrap 5, con personalizaciones clave integradas directamente en etiquetas <style></style> dentro de cada archivo HTML.

Estilo Clave Unificado:

  • Navbar superior de color rojo institucional (#ab0a0a)
  • Fondo general de página gris claro (bg-light o #f4f6f9)
  • Uso extensivo de componentes de Bootstrap como tarjetas (.card) con encabezados blancos y sombras sutiles para organizar el contenido
  • Tipografía Poppins para main.html
  • Iconografía de Font Awesome
  • Layout responsivo gracias a Bootstrap

  • Manejo del formulario de inicio de sesión en index.html (guardado de datos en localStorage)
  • Función checkLogin() para verificar la sesión en main.html y mostrar el nombre de usuario

Funcionalidad Principal:

Las páginas perfil_usuario.html, editar-usuario.html, editar-identificaciones.html, editar-academica.html, y editar-contactos.html contienen JavaScript embebido para:

  • Guardar los datos modificados de vuelta a localStorage
  • Manejar la interactividad de elementos dinámicos (ej. añadir/quitar idiomas, títulos, direcciones)

7. Resumen General de Mejoras Implementadas

  • Diseño Unificado y Moderno: Se estableció una paleta de colores consistente (rojo institucional #ab0a0a, blanco y grises claros) y un estilo de tarjetas (.card con sombra) para la mayoría de las vistas internas, utilizando Bootstrap 5 como base
  • Consistencia Visual: El navbar superior y el footer son consistentes en todas las páginas internas
  • Mejora de la Experiencia de Usuario (UX):
    • Navegación más clara e intuitiva
    • Formularios mejor estructurados y más fáciles de usar (ej. uso de <select> en lugar de inputs de texto)
    • Organización de la información en secciones lógicas mediante tarjetas
    • Diseño responsivo para una mejor visualización en diferentes dispositivos
  • Claridad en la Presentación de Información: Espaciado mejorado y uso de iconografía para facilitar la comprensión
  • Funcionalidad de Perfil Mejorada: Las vistas de perfil y edición son más robustas y permiten una gestión detallada de la información del usuario con persistencia en localStorage
  • Panel Principal (Dashboard) Rediseñado: La vista main.html ahora presenta un diseño más profesional con sidebar, header y contenido principal bien diferenciados, incluyendo un slider visual y tarjetas de avisos
